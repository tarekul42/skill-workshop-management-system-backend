# Security Policy

## Supported Versions

The following versions of the project are currently supported with security updates.

| Version | Supported          |
| ------- | ------------------ |
| 1.0.x   | :white_check_mark: |
| < 1.0   | :x:                |

## Reporting a Vulnerability

We take the security of our software seriously. If you have found a vulnerability, please report it to us immediately.

### How to Report

Please do NOT report security vulnerabilities through public GitHub issues.

Instead, please send an email to **tarekulrifat142@gmail.com**.

In your report, please include:

- A clear description of the vulnerability.
- Steps to reproduce the issue (including any scripts or code).
- The potential impact of the vulnerability.

We will acknowledge receipt of your report within 48 hours and will provide a timeline for the fix.

## Response Timeline

- **Initial Response**: Within 48 hours.
- **Status Update**: Every 5 business days until resolved.
- **Resolution**: We aim to resolve critical vulnerabilities within 14 days.

## Out of Scope

The following are generally strictly out of scope:

- Denial of Service (DoS) attacks.
- Clickjacking on pages with no sensitive actions.
- Social Engineering / Phishing.
